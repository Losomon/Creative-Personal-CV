# Contributing

Thank you for your interest in contributing to this portfolio repository.

## Before you begin

Please read this guide before submitting issues or pull requests. Also review `CODE_OF_CONDUCT.md` to understand the community expectations.

## Getting started

1. Fork the repository.
2. Create a branch with a descriptive name, for example: `feature/update-portfolio` or `fix/hero-section`.
3. Make your changes in the `frontend` folder when working on the portfolio site.
4. Commit with a clear message.
5. Open a pull request against the `main` branch.

## Local development

Install dependencies:

```bash
cd frontend
npm install
```

Run the app locally:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

## Suggested workflow

- Open an issue first for larger features or design changes.
- Keep pull requests small and focused.
- Reference the issue number in your PR description.
- Update documentation when behavior or UI changes.

## PR expectations

- Keep changes targeted to a single purpose.
- Add or update documentation when necessary.
- Verify `npm run build` succeeds in `frontend`.
- Ensure code follows the existing style and formatting.

## Issue and PR templates

This repository includes issue templates in `.github/ISSUE_TEMPLATE/` for bugs, features, and questions. Please use those templates when opening a new issue.

## Code style

- Follow the existing React and CSS conventions.
- Keep markup semantic and accessible.
- Use clear, descriptive names for components and variables.

## Community and support

If you have questions or need guidance, open an issue or leave a comment on an existing discussion. We welcome contributions from all backgrounds.
